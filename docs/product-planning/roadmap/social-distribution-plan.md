# Local Memory Hub 社交传播方案

## 核心矛盾与策略
本产品本地优先、强隐私,直接"分享数据"违背本质。社交的正确姿势是:
**把私密的记忆加工成可炫耀、可安全外发的视觉产物,每个产物带水印 + 项目链接 → 自然引流。** 开源 + 本地 + MCP 本身就是极客圈传播点。

## P0 — 立刻能做(数据/能力现成)
| 功能 | 做什么 | 为什么传播 | 复用 |
| --- | --- | --- | --- |
| ① 记忆报告卡(类 Wrapped) | 一张卡:已沉淀 X 条记忆 / Y 节点 / Z 问答,关注 Top 关键词 | 个人 stats 卡是被验证的刷屏格式 | `/api/memory/habits` + 计数 + `/api/graph/export` |
| ② 知识图谱海报 | 图谱快照包装成带标题/统计/水印的海报 | 图谱是社交货币 | 已有图谱导出 |
| ③ 洞察金句卡 | 把一条问答回答导出成带引用的引用卡 | 金句易转发 | QA 回答 + citations |

共性:输出**图片**(不含原文)+ 右下角品牌水印 + GitHub 链接。零隐私泄露、强引流。

## P1 — 中期
- ④ 隐私安全分享:分享前自动脱敏,只导出选中主题簇/单条。
- ⑤ 配方/模板市场:分享不含数据的连接器配方、embedding 选型、提示词模板。
- ⑥ 可嵌入只读图谱:静态 HTML/图片贴进博客/Notion。

## P2 — 远期

- ⑧ 图谱视觉升级(多主题皮肤、动效、高质量分享渲染)。
- ⑨ 完整社交网络(PRD 明确 V1 不做)。
