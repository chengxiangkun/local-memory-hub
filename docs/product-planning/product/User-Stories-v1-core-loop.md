# 用户故事：Local Memory Hub V1 核心闭环

## 1. 范围

V1 的 6 个核心卖点全部必须覆盖：

1. 先让资料可信地进入系统。
2. 再让资料变成可用记忆。
3. 再让用户能搜索和问答。
4. 再让用户能治理污染。
5. 再让图谱成为探索和传播点。
6. 最后让外部 AI 工具安全调用。

本文件把这些核心卖点拆成 Epic、用户故事和验收标准。

优先级说明：

- P0：V1 必须完成，否则核心闭环断裂。
- P1：V1 应完成，影响体验完整度。
- P2：可后置到 Beta 或后续版本。

## 2. 用户角色

| 角色 | 说明 |
| --- | --- |
| 普通知识工作者 | 导入文件、网页、截图、外部文档，并搜索使用 |
| AI 重度用户 | 使用问答、模型配置、语义检索和图谱 |
| 开发者用户 | 使用 Codex/MCP/API 调用本地记忆 |
| 治理型用户 | 关心污染、删除、追溯和隐私 |

## 3. Epic 1：资料可信地进入系统

### US-1.1 导入本地文件

**优先级：P0**

**描述：** 作为用户，我希望可以上传或拖拽本地文件，以便系统先保存源文件并创建可追溯记录。

**验收标准：**

1. 用户可以通过上传按钮或拖拽导入文件。
2. 导入成功后，系统创建 `SourceRecord`。
3. 源资料库显示文件名、类型、导入时间、本地路径和导入状态。
4. 导入完成不代表解析完成，解析状态应单独显示。
5. 如果文件重复，系统提示已存在并支持合并。
6. 如果导入失败，系统显示可理解的失败原因。

**依赖：** SourceRecord、本地文件存储、导入状态机。

### US-1.2 粘贴链接导入

**优先级：P0**

**描述：** 作为用户，我希望粘贴网页或视频链接，以便系统保存来源并尝试获取基础信息。

**验收标准：**

1. 用户可以在工作台或 Inbox 粘贴 URL。
2. 系统保存原始 URL 和 canonical URL。
3. 系统识别来源平台，如通用网页、B 站、YouTube、飞书、有道云。
4. 如果无法直接解析，系统仍保存链接，并提示后续操作。
5. 源资料库中可按来源平台筛选该链接。

**依赖：** UrlImporter、ImporterRegistry、SourceRecord。

### US-1.3 快捷键保存剪贴板内容

**优先级：P0**

**描述：** 作为用户，我希望用快捷键保存剪贴板内容，以便在不切换工作流的情况下收集资料。

**验收标准：**

1. 用户可以配置或使用默认快捷键。
2. 触发快捷键后，系统读取剪贴板文本或图片。
3. 系统生成 `ImportRequest`，不得绕过导入管线。
4. 导入成功后显示轻量通知。
5. 资料出现在 Inbox 和源资料库中。

**依赖：** ClipboardImportCommand、ClipboardImporter、Import Pipeline。

### US-1.4 截图导入

**优先级：P0**

**描述：** 作为用户，我希望截图后直接导入系统，以便保存聊天、网页、PPT 或代码截图。

**验收标准：**

1. 用户可以通过快捷键或按钮截图。
2. 系统保存原始图片文件。
3. 系统创建源资料记录。
4. 解析状态初始为等待 OCR。
5. 截图可以按图片类型、日期和来源筛选。

**依赖：** ScreenshotImportCommand、ScreenshotImporter、本地文件存储。

### US-1.5 导入飞书文档

**优先级：P0**

**描述：** 作为用户，我希望导入飞书文档，以便把外部文档纳入本地源资料库。

**验收标准：**

1. 用户可以粘贴飞书文档链接。
2. 系统识别为飞书来源。
3. 如果有授权，系统尝试通过连接器拉取文档内容。
4. 如果无法拉取，系统保存链接并提示授权、导出或上传兜底。
5. 源资料记录保存飞书链接、标题、平台信息和导入状态。

**依赖：** FeishuImporter、FeishuClient、外部连接器设置。

### US-1.6 连接腾讯文档

**优先级：P0**

**描述：** 作为用户，我希望连接腾讯文档，以便把常用在线文档同步到本地源资料库。

**验收标准：**

1. 用户可以粘贴腾讯文档链接。
2. 系统识别来源为腾讯文档。
3. 如果有授权，系统尝试通过连接器拉取文档内容。
4. 系统保存文档 ID、标题、来源链接、授权账号和最近同步时间。
5. V1 支持手动“立即同步”和定时轮询同步。
6. 如果授权失效或 API 受限，系统保存链接并提示导出/上传兜底。

**依赖：** TencentDocImporter、TencentDocClient、TencentDocAuthProvider、同步调度器。

### US-1.7 导入有道云笔记

**优先级：P0**

**描述：** 作为用户，我希望导入有道云笔记，以便把已有笔记纳入本地资料库。

**验收标准：**

1. 用户可以粘贴有道云链接、上传导出文件或复制内容导入。
2. 系统识别来源为有道云。
3. 系统保存标题、来源链接和导入方式。
4. 如果平台访问受限，系统给出导出/复制内容兜底提示。
5. 导入结果进入 Inbox。

**依赖：** YoudaoImporter、YoudaoClient、手动兜底流程。

### US-1.8 外部文档同步状态可见

**优先级：P0**

**描述：** 作为用户，我希望知道每个外部文档是否已连接、是否同步成功、是否授权失效，以便降低重复手动导入成本。

**验收标准：**

1. 源资料库展示同步状态，如未开启同步、已连接、同步中、同步成功、同步失败、授权失效、需要导出。
2. 用户可以手动触发“立即同步”。
3. 外部文档发生变化后，系统生成新的源资料版本，不覆盖旧版本。
4. 新版本重新进入解析、向量和图谱流水线。
5. 授权失效时，普通搜索仍可使用最后一次成功同步的本地快照。

**依赖：** ConnectorSyncScheduler、SourceRevision、外部连接器设置。

### US-1.9 源资料状态可见

**优先级：P0**

**描述：** 作为用户，我希望看到每条资料的导入状态、解析状态、入记忆状态和污染状态，以便判断资料是否可用。

**验收标准：**

1. 源资料库列表显示导入状态。
2. 源资料库列表显示解析状态。
3. 源资料库列表显示是否已入记忆。
4. 源资料库列表显示污染/隔离状态。
5. 点击状态可以查看详情和失败原因。
6. 状态变化后页面自动刷新或给出通知。

**依赖：** 状态机、SourceRecord、任务事件。

## 4. Epic 2：资料变成可用记忆

### US-2.1 本地解析常见文件

**优先级：P0**

**描述：** 作为用户，我希望系统本地解析常见文件，以便在不依赖外部模型的情况下提取内容。

**验收标准：**

1. 系统支持 PDF、文本、Markdown、图片、音频/视频基础转写的本地解析入口。
2. 解析任务异步执行，不阻塞导入。
3. 解析成功后保存抽取文本。
4. 解析失败后保存失败原因。
5. 用户可以重新解析。

**依赖：** Parser Pipeline、Parse Job、本地 worker。

### US-2.2 本地解析失败后调用大模型兜底

**优先级：P0**

**描述：** 作为用户，我希望本地解析失败或质量低时，系统可以调用我配置的大模型，以便尽量得到可用内容。

**验收标准：**

1. 当本地解析失败或置信度低时，系统进入大模型兜底状态。
2. 系统根据用户模型策略选择供应商和模型。
3. 调用前记录来源、模型、任务目的和内容大小。
4. 兜底成功后保存大模型解析结果。
5. 兜底失败后显示失败原因，并允许重试。

**依赖：** ModelRouter、ProviderAdapter、调用日志。

### US-2.3 大模型解析结果反哺本地解析规则

**优先级：P0**

**描述：** 作为用户，我希望大模型修正结果可以帮助系统下次更好地本地解析，以便减少后续 token 消耗。

**验收标准：**

1. 系统保存本地解析失败输出和大模型修正输出。
2. 系统生成解析改进产物，如规则、模板或样例。
3. 系统记录规则适用来源类型和置信度。
4. 下次同类内容优先尝试该规则。
5. 系统记录该规则的成功次数和失败次数。

**依赖：** Parser Improvement Artifact、规则存储、解析调度器。

### US-2.4 内容切分并进入向量记忆

**优先级：P0**

**描述：** 作为用户，我希望解析后的内容被切分并进入向量记忆，以便后续可以语义搜索和问答。

**验收标准：**

1. 解析文本按结构或长度切分为 chunk。
2. 每个 chunk 保留源资料 ID 和位置。
3. 系统为 chunk 生成 embedding。
4. 入记忆成功后更新 memory_status。
5. 入记忆失败后显示失败原因并支持重试。

**依赖：** Chunker、Embedding Service、Memory Index。

### US-2.5 记忆可追溯回源文件

**优先级：P0**

**描述：** 作为用户，我希望每条记忆都能追溯回源文件，以便验证 AI 使用的信息来源。

**验收标准：**

1. 每个 chunk 关联 source_id。
2. 记忆库中可以打开对应源文件。
3. 对视频/音频类资料，记忆应记录时间位置。
4. 对 PDF 类资料，记忆应记录页码或位置。
5. 如果源文件已删除，系统显示当前追溯状态。

**依赖：** SourceRecord、Chunk、Trace Metadata。

## 5. Epic 3：搜索和问答

### US-3.1 无大模型时搜索可用

**优先级：P0**

**描述：** 作为用户，我希望即使没有配置大模型，也能搜索资料，以便产品不会因为模型配置缺失而不可用。

**验收标准：**

1. 未配置大模型时，全局搜索仍可使用。
2. 系统返回源资料、命中片段和相关记忆。
3. 页面明确提示当前为搜索兜底，不是 AI 总结答案。
4. 用户可以从结果打开源文件。
5. 污染或隔离内容默认不出现在普通搜索结果中。

**依赖：** FullTextRetriever、SourceRetriever、Search UI。

### US-3.2 有大模型时生成带引用答案

**优先级：P0**

**描述：** 作为用户，我希望配置大模型后可以基于本地记忆问答，并看到答案引用来源。

**验收标准：**

1. 用户提问后，系统先检索相关上下文。
2. 系统调用配置的大模型生成答案。
3. 答案必须包含引用来源。
4. 用户可以点击引用回到源资料。
5. 如果没有足够资料，系统应说明资料不足，而不是编造。

**依赖：** QueryPlanner、Retriever、ModelRouter、Citation Renderer。

### US-3.3 图谱参与问答召回

**优先级：P1**

**描述：** 作为用户，我希望问答时不仅使用向量召回，也能结合图谱扩展相关内容，以便回答更完整。

**验收标准：**

1. 系统先进行向量召回。
2. 系统将召回 chunk 映射到图谱节点。
3. 系统沿图谱扩展一跳或两跳。
4. 污染节点和隔离节点不参与扩展。
5. 问答结果展示使用到的图谱节点。

**依赖：** GraphRetriever、VectorRetriever、Result Fusion。

### US-3.4 用户标记错误引用

**优先级：P0**

**描述：** 作为用户，我希望可以标记回答中的错误引用，以便系统后续不再使用错误资料。

**验收标准：**

1. 每条引用旁边有标记错误或污染入口。
2. 用户标记后，系统记录原因。
3. 被标记内容进入污染或待确认状态。
4. 后续普通问答不再默认使用该内容。
5. 用户可以在治理页面恢复。

**依赖：** Pollution State、Answer Citation、Governance UI。

## 6. Epic 4：污染治理

### US-4.1 标记污染并隔离

**优先级：P0**

**描述：** 作为用户，我希望可以把错误、过期或不想让 AI 使用的内容标记为污染，以便它不再影响搜索和问答。

**验收标准：**

1. 用户可以在源资料、记忆库、问答引用、图谱节点中标记污染。
2. 用户可以选择污染原因。
3. 标记后内容进入隔离状态。
4. 隔离内容默认不参与搜索、问答和图谱推荐。
5. 用户可以查看隔离列表。

**依赖：** Pollution State Machine、Search Filter、Graph Filter。

### US-4.2 删除源文件时选择删除范围

**优先级：P0**

**描述：** 作为用户，我希望删除源文件时可以选择是否删除解析文本、向量和图谱，以便控制数据生命周期。

**验收标准：**

1. 用户删除源文件时，系统弹出删除选项。
2. 选项包括仅删除源文件、删除解析文本、删除向量和图谱、全部删除、移入回收站。
3. 默认选项为移入回收站并从检索排除。
4. 系统执行删除后更新追溯状态。
5. 删除操作写入删除记录。

**依赖：** Deletion Policy、Derived Artifact Registry、Trash。

### US-4.3 恢复隔离内容

**优先级：P1**

**描述：** 作为用户，我希望可以恢复被隔离的内容，以便误操作不会造成永久损失。

**验收标准：**

1. 用户可以查看隔离内容列表。
2. 用户可以恢复单条或多条内容。
3. 恢复后内容重新参与搜索和问答。
4. 如果向量或图谱已删除，系统提示需要重建。
5. 恢复操作写入记录。

**依赖：** Quarantine、Restore Workflow、Rebuild Jobs。

## 7. Epic 5：图谱探索和传播

### US-5.1 查看 Obsidian 风格图谱

**优先级：P0**

**描述：** 作为用户，我希望看到资料、主题、想法和人物之间的图谱关系，以便探索自己的信息网络。

**验收标准：**

1. 图谱显示源文件、主题、人物、项目、想法等节点。
2. 图谱支持缩放和平移。
3. 图谱支持节点拖拽。
4. 节点 hover 时高亮自身和相关关系。
5. 选中节点后高亮一跳关系。
6. 节点根据类型使用不同视觉样式。
7. 点击节点打开详情。
8. 节点详情可跳转源资料或记忆。
9. 图谱加载时有轻量节点弹出动效。
10. 设置页支持根据已有文本片段重建图谱索引。
11. 重建图谱时，不修改源文件、文本片段和向量索引。
12. 不同资料命中相同关键词时，应共享关键词节点并形成跨资料关联。

**V1 不包含：** 节点跟随鼠标移动、靠近避让、局部力场扰动。

**依赖：** Graph Store、Graph UI、Node Detail Panel。

### US-5.2 图谱搜索和节点定位

**优先级：P0**

**描述：** 作为用户，我希望在图谱页搜索节点或关系，以便快速定位内容。

**验收标准：**

1. 图谱页提供搜索框。
2. 用户可以搜索节点名称、主题、人物、项目、源文件。
3. 命中节点高亮。
4. 图谱自动聚焦到命中节点。
5. 右侧面板显示节点详情和相邻节点。

**依赖：** Graph Search、Graph UI。

### US-5.3 过滤污染节点

**优先级：P0**

**描述：** 作为用户，我希望图谱可以隐藏污染或隔离节点，以便图谱不会被错误数据干扰。

**验收标准：**

1. 图谱默认不显示隔离节点。
2. 用户可以切换显示污染节点。
3. 污染节点有明显视觉标记。
4. 用户可以从图谱中移除污染节点或关系。
5. 移除后不影响源文件，除非用户选择删除派生产物。

**依赖：** Graph Filter、Pollution State。

### US-5.4 导出图谱快照

**优先级：P1**

**描述：** 作为用户，我希望导出图谱快照，以便分享自己的知识关系或阶段成果。

**验收标准：**

1. 用户可以导出当前图谱视图为图片。
2. 导出时可以选择隐藏敏感来源。
3. 导出内容不包含本地路径等敏感信息，除非用户确认。
4. 导出后保留当前筛选和聚焦状态。
5. 失败时显示原因。

**依赖：** Graph Renderer、Export Service、Privacy Options。

## 8. Epic 6：外部 AI 工具安全调用

### US-6.1 启用本地 MCP Server

**优先级：P0**

**描述：** 作为开发者用户，我希望启用本地 MCP Server，以便 Codex 或其他 AI 工具可以读取授权的本地记忆。

**验收标准：**

1. 用户可以在设置中启用 MCP Server。
2. 系统显示本地连接信息。
3. 用户可以查看可用 MCP 工具列表。
4. MCP 默认只监听本地地址。
5. 关闭 MCP 后外部工具无法继续调用。

**依赖：** MCP Server、Settings UI、Permission Layer。

### US-6.2 外部工具搜索本地记忆

**优先级：P0**

**描述：** 作为开发者用户，我希望 Codex 可以搜索本地记忆，以便写代码或文档时不用手动复制上下文。

**验收标准：**

1. 外部工具可以调用 `memory.search`。
2. 搜索结果包含标题、片段、来源和状态。
3. 污染或隔离内容默认不返回。
4. 调用记录写入日志。
5. 用户可以在应用内查看最近外部调用。

**依赖：** MCP Tools、Search Service、Call Logs。

### US-6.3 外部工具获取上下文

**优先级：P0**

**描述：** 作为开发者用户，我希望外部 AI 工具可以获取结构化上下文，以便由外部模型自己生成答案。

**验收标准：**

1. 外部工具可以调用 `memory.get_context`。
2. 返回内容包含 chunk、引用来源和追溯信息。
3. 返回内容受用户权限和过滤规则限制。
4. 返回内容不包含隔离数据。
5. 调用日志记录请求来源和使用范围。

**依赖：** Context Builder、Permission Layer、MCP。

### US-6.4 外部导入资料

**优先级：P1**

**描述：** 作为开发者用户，我希望外部工具可以通过 API/MCP 导入资料，以便自动把工作流中的资料加入系统。

**验收标准：**

1. 外部工具可以调用 `source.import`。
2. 请求会生成标准 `ImportRequest`。
3. 外部导入不能绕过导入管线。
4. 导入结果返回 source_id 和状态。
5. 用户可以在源资料库看到该外部导入来源。

**依赖：** Import API、MCP Import Tool、Import Pipeline。

## 9. V1 完整闭环验收

V1 完整闭环必须满足：

1. 用户导入一份 PDF、一张截图、一个网页链接、一个飞书文档和一个有道云内容。
2. 系统创建源资料记录，并显示状态。
3. 系统本地解析资料。
4. 本地解析失败时，系统调用用户配置的大模型兜底。
5. 解析结果进入向量记忆。
6. 用户不配置大模型时，可以搜索资料并看到命中片段。
7. 用户配置大模型后，可以问答并看到引用。
8. 用户可以把一条错误记忆标记为污染。
9. 污染内容不再参与普通搜索、问答和图谱推荐。
10. 用户可以在图谱中搜索节点并回到源资料。
11. Codex 或外部 AI 工具可以通过 MCP 搜索本地记忆。
12. 所有外部调用有日志和权限控制。

## 10. 后续建议

下一步应使用 `prioritization-frameworks`，对这些用户故事做 P0/P1/P2 之外的更细排序。

推荐只先排序 P0：

- 先保证源资料进入和状态可信。
- 再保证解析和入记忆。
- 再保证无模型搜索和有模型问答。
- 再保证污染隔离。
- 再保证图谱搜索。
- 最后保证 MCP 搜索和上下文获取。
