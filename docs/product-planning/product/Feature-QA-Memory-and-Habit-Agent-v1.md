# 功能说明：问答记忆沉淀与记忆整理 Agent

## 功能目标

问答页面不是一次性对话窗口。每次用户向本地记忆提问后，系统应把有价值的问答过程加工成可追溯的本地记忆，再进入文本片段、向量索引和图谱流水线。

同时，系统需要一个本地优先的记忆整理 Agent，维护用户习惯画像，为后续个性化检索、推荐和低 token 消耗策略提供基础。

## 用户价值

- 对话产生的新结论不会丢失。
- 每条问答记忆都可以像源资料一样被搜索、追溯、隔离和删除。
- 系统能逐步理解用户常问主题、常用资料类型和偏好，不需要每次重新解释背景。
- 画像计算优先使用本地统计，默认不消耗外部模型 token。

## V1 范围

### 问答自动沉淀

每次 `/api/ask` 成功生成回答后，默认创建一条 `qa_conversation` 源资料。

该源资料包含：

- 用户问题
- 模型或本地兜底回答
- 引用来源
- 生成时间
- 记忆整理提示

随后复用现有主流水线：

1. 保存为本地源资料。
2. 本地解析为文本。
3. 切分为文本片段。
4. 写入向量索引。
5. 生成图谱节点与关系。
6. 支持污染隔离和恢复。

### 记忆整理 Agent

V1 先实现确定性、本地、低成本版本。

Agent 读取本地问答记忆，输出用户习惯画像：

- 已分析的问答记忆数量
- 高频关键词
- 可能的偏好，例如本地优先、图谱探索、低 token 成本、导入解析质量
- 本地优先、可治理、可删除策略说明

## 非目标

V1 不做：

- 自动向外部模型发送画像。
- 复杂心理画像。
- 多用户画像合并。
- 后台定时任务。
- 基于画像自动改写源资料或图谱关系。

## 数据流

```text
用户问题
  -> 本地搜索 / 向量检索 / 图谱检索
  -> 模型或本地兜底回答
  -> 生成问答记忆源资料
  -> 本地解析
  -> 文本片段
  -> 向量索引
  -> 图谱节点
  -> 记忆整理 Agent 统计习惯画像
```

## 治理要求

- 问答记忆必须显示为源资料，不能成为不可见的黑盒向量。
- 用户可以搜索、隔离、恢复或删除问答记忆。
- 被隔离的问答记忆不得参与搜索、向量召回和图谱展示。
- 画像必须可重新生成；删除画像不应删除源资料和记忆。

## API

### `POST /api/ask`

默认在返回回答时附带：

```json
{
  "conversation_memory": {
    "status": "persisted",
    "source_id": "...",
    "segment_count": 1,
    "graph_node_count": 2
  }
}
```

如请求体设置：

```json
{
  "persist_memory": false
}
```

则本次问答不进入记忆。

### `GET /api/memory/habits`

返回本地记忆整理 Agent 生成的用户习惯画像。

## 验收标准

- 完成一次问答后，源资料库新增一条 `qa_conversation` 记录。
- 该记录已解析、已入记忆，并能被全文搜索命中。
- 该记录写入向量索引，能被 `/api/vector/search` 召回。
- 图谱中出现该问答记忆相关节点。
- `GET /api/memory/habits` 返回问答记忆数量、高频关键词和偏好推断。
- 请求 `POST /api/ask` 时设置 `persist_memory: false`，不会新增问答记忆。
