# 功能说明：中英文混合检索与向量偏移校准

## 1. 背景

用户资料天然会混合中文、英文、缩写、代码名、模型名和平台名。只做中文关键词会漏掉英文表达；只做向量检索会遇到中英文 embedding 空间偏移、专有名词召回不稳、短查询漂移等问题。

## 2. V1 目标

- 中文、英文和中英混合查询都能召回相关文本片段。
- 专有名词、模型名、API 名称、项目名优先稳定命中。
- 向量检索不单独决定最终排序，必须有本地词面校准。
- 默认不为检索调用大模型，节省 token。

## 3. 检索策略

V1 使用混合检索：

```text
用户查询
  -> 中英文边界归一化
  -> 英文 token
  -> 中文 2-gram
  -> 本地向量检索
  -> 词面重排
  -> 融合分数排序
```

融合公式：

```text
hybrid_score = vector_score * 0.75 + lexical_score * 0.25
```

## 4. 中文处理

- 中文连续文本生成 2-gram。
- 2 到 4 个字的中文短词整体保留。
- 常见停用词过滤。
- 不在 UI 中展示 `chunk`、`embedding` 等英文术语。

## 5. 英文处理

- 英文统一小写。
- 保留长度大于等于 2 的 token。
- 过滤基础停用词。
- 模型名、API 名、项目名按普通 token 进入索引。

## 6. 验收标准

1. 查询 `语义搜索` 能召回包含“语义搜索”的文本片段。
2. 查询 `AI memory graph 记忆图谱` 能召回包含“AI记忆图谱”和 `graph memory` 的文本片段。
3. 检索结果返回 `vector_score` 和 `lexical_score`，便于后续调试排序。
4. 污染或隔离资料不参与普通向量检索。
5. 后续替换真实 embedding 模型时，仍保留 lexical rerank。
