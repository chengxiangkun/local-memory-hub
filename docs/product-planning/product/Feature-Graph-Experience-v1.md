# 功能规格：图谱探索体验 V1

## 1. 目标

图谱页是 Local Memory Hub 的默认首页，也是产品的第一视觉记忆点。

V1 目标不是做一个完整图谱编辑器，而是让用户能快速感受到：

```text
我的源资料、主题、记忆和想法正在形成一个可以探索、可以追溯、可以治理的网络。
```

## 2. 当前设计判断

Obsidian 风格图谱不是一个可直接复用的开源框架。它更像一种体验方向：

- 深色沉浸画布。
- 节点和关系形成自然网络。
- 支持缩放、平移、拖拽。
- 搜索后能定位节点。
- hover 或选中时能高亮相关路径。
- 节点详情可以追溯源资料。

“节点跟随鼠标移动、靠近鼠标自动避让、局部力场扰动”不是 Obsidian 基础能力，不进入 V1 必须范围。

## 3. V1 必须实现

### 3.1 图谱画布

- 默认首页打开图谱页。
- 显示源资料、外部文档、主题、实体、项目、人物、想法、问题、记忆等节点。
- 显示节点之间的关系边。
- 节点按类型使用不同视觉样式。
- 关系必须能解释来源。
- 污染或隔离节点默认不显示。

### 3.2 基础交互

- 缩放。
- 拖拽旋转 3D 球形图谱视角。
- 节点 hover 高亮。
- 节点点击选中。
- 选中节点后高亮一跳关系。
- 点击节点时懒加载一跳邻居，不刷新页面。
- 右侧详情面板展示节点详情。
- 节点详情可以打开源资料或关联记忆。

### 3.3 搜索定位

- 图谱页提供搜索入口。
- 支持搜索节点名称、主题、人物、项目、源文件。
- 命中节点高亮。
- 图谱自动聚焦到命中节点。
- 搜索为空时恢复默认视图。

### 3.4 治理联动

- 隔离源资料后，图谱节点和关系应从普通视图中隐藏。
- 用户可切换查看污染/隔离节点。
- 污染节点必须有明显视觉标记。
- 删除源资料时，用户可以选择是否删除图谱节点和关系。

### 3.5 大图性能策略

V1 图谱不能默认全量渲染。节点多时按分层方式进入：

```text
默认概览：最多加载 200 个最近节点
  ↓
社区概览：按 node_type 聚合为源资料、主题、关键词、记忆等社区节点
  ↓
类型下钻：点击社区节点后加载该类型代表节点和少量邻居
  ↓
节点展开：点击普通节点后加载一跳邻居
  ↓
搜索子图：搜索时只加载命中节点及其邻居
```

当前接口边界：

- `GET /api/graph?limit=200`：默认概览。
- `GET /api/graph/communities`：社区聚合图。
- `GET /api/graph/type?node_type=keyword&limit=80`：类型子图。
- `GET /api/graph/neighbors?node_id=...&limit=120`：节点邻居。
- `GET /api/graph/subgraph?q=...`：搜索子图。

这些接口是后续替换为 Force Graph、D3 或 Sigma.js 时的稳定数据边界。

## 4. V1 不做

- 完整图谱编辑器。
- 多人协作图谱。
- 完整社交网络。
- 节点实时跟随鼠标移动。
- 鼠标靠近节点自动避让。
- 局部力场扰动和复杂物理交互。
- 大规模图谱 WebGL 优化，除非 Spike 证明 V1 必须。

## 5. 后置增强

这些能力可以进入 V1 后段或 V2：

- 节点跟随鼠标轻微响应。
- 鼠标靠近节点自动避让。
- 局部力场扰动。
- 关系路径动画。
- 用户手动创建、编辑和锁定节点关系。
- 图谱快照模板和分享海报。

这些增强必须满足：

- 可关闭。
- 支持 `prefers-reduced-motion`。
- 不影响节点点击和搜索定位。
- 不因高频交互造成卡顿。

## 6. 技术选型原则

正式实现前必须先做图谱库 Spike，避免重复造轮子。

优先验证：

| 方案 | 适合点 | 风险 | 参考 |
| --- | --- | --- |
| Force Graph / 2D Force Graph | 接近 Obsidian 的 Canvas 力导向、缩放、拖拽、hover/click 体验 | 需要验证打包体积、样式可控性 | https://github.com/vasturiano/force-graph |
| D3 force | 自定义能力强，适合 SVG/Canvas 混合 | 工程实现复杂度较高 | https://d3js.org/d3-force |
| Sigma.js | 大规模图谱和 WebGL 友好 | 对 V1 可能偏重 | https://www.sigmajs.org/ |
| Cytoscape.js | 图分析和交互成熟 | 视觉风格可能需要较多定制 | https://js.cytoscape.org/ |

当前静态 SVG 原型只作为体验验证和 fallback 参考，不作为最终图谱架构承诺。

## 7. 渲染层边界

图谱渲染层只消费结构化数据：

```text
GraphViewModel
  nodes[]
  edges[]
  selectedNodeId
  matchedNodeIds
  filters
  viewportState
```

渲染层不得直接负责：

- 导入逻辑。
- 解析逻辑。
- 向量检索。
- 污染治理业务规则。
- 数据删除策略。

这样后续从 SVG 原型替换为 Force Graph、D3 force 或 Sigma.js 时，不需要重写业务流程。

## 8. 验收标准

V1 图谱可验收需要同时满足：

1. 默认 200 个节点以内交互可用。
2. 支持缩放、平移和拖拽。
3. 搜索能高亮并定位节点。
4. 点击节点能打开详情。
5. 详情能追溯源资料。
6. 隔离节点默认不进入图谱视图。
7. 污染节点可切换显示并有明显标记。
8. 图谱加载失败时提供列表兜底。
9. 动效不影响可读性。
10. 减少动态设置开启时，非必要动画停止。
11. 大图通过社区概览、类型下钻、搜索子图和邻居懒加载避免一次性全量渲染。

## 9. 开发前置条件

进入正式代码开发前，至少完成：

- 图谱库 Spike。
- GraphViewModel 数据结构确认。
- 图谱搜索 API 确认。
- 节点详情 API 确认。
- 污染过滤和删除级联规则确认。
- 首屏性能预算确认。
