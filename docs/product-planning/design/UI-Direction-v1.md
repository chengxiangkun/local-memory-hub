# UI 设计方向：Local Memory Hub V1

## 1. 设计读法

这是一个本地优先 AI 记忆桌面产品，面向开发者、AI 重度用户和高信息密度知识工作者。

整体风格应接近：

- Obsidian 的图谱探索感
- Raycast 的快捷和效率
- Linear 的克制、清晰和状态表达
- 开发工具的可信、稳定和可追溯

它不是营销网站，也不是纯聊天 App。它应该像一个安静但有生命感的本地记忆工作台。

## 2. 第一印象

应用打开后的默认首页是图谱首页。

第一眼要传达：

```text
我的资料正在形成一个可以探索的记忆网络。
```

不是：

```text
这里有一个后台管理系统。
```

## 3. 首次启动体验

安装后必须有引导。

引导目标不是讲解所有功能，而是让用户完成第一次成功体验。

引导步骤：

1. 选择本地数据目录。
2. 配置大模型，可跳过。
3. 导入一段文本。
4. 系统生成第一条源资料、第一条记忆和第一个图谱节点。
5. 进入图谱首页。

引导语气要轻：

```text
随便粘贴一段你最近看到的内容。
```

不要让用户一开始就面对复杂配置。

## 4. 首页图谱方向

图谱首页应像 Obsidian，但需要更产品化：

- 图谱是主画布。
- 搜索和快捷导入常驻顶部。
- 右侧详情面板展示节点信息。
- 默认隐藏污染节点。
- 空状态也要保留图谱感，但不能伪造数据。

图谱视觉可以更有冲击力：

- 深色画布优先。
- 节点有类型差异。
- 选中节点有路径高亮。
- 聚类之间有空间层级。
- 节点移动和缩放要顺滑。
- V1 优先实现拖拽、缩放、平移、搜索定位、hover/选中高亮。
- “节点跟随鼠标移动/靠近避让”不是 Obsidian 基础能力，不作为 V1 必须项，后续作为高级图谱互动增强验证。

但必须保证：

- 字能看清。
- 状态能看懂。
- 能搜索。
- 能点回源文件。

## 5. 中文术语

面向用户的界面必须优先使用中文术语。

### 推荐术语

| 技术词 | 用户界面用词 |
| --- | --- |
| chunk | 文本片段 |
| embedding | 向量索引 |
| vector search | 语义搜索 |
| parser | 解析器 |
| parse job | 解析任务 |
| source record | 源资料记录 |
| source | 源资料 |
| MCP Server | MCP 本地服务 |
| provider | 模型供应商 |
| endpoint | 接口地址 |
| fallback | 兜底 |

### 可保留英文的情况

这些词可以保留英文，因为目标用户能理解，且翻译后反而别扭：

- API
- MCP
- Codex
- OAuth
- OpenAI-Compatible
- Anthropic-Compatible

## 6. 状态视觉

状态需要克制但清楚。

### 状态分层

```text
已保存
解析中
解析成功
解析失败
已入记忆
未入记忆
已污染
已隔离
可追溯
追溯异常
```

### 颜色建议

- 成功：绿色系，低饱和。
- 处理中：蓝色或青色系。
- 需要操作：琥珀色。
- 失败：红色，但不要刺眼。
- 污染/隔离：紫红或深红，和普通失败区分。
- 未配置/未入记忆：中性色。

不要做大面积紫蓝 AI 渐变。图谱可以有光效，但基础 UI 要克制。

## 7. 页面气质

### 普通页面

源资料、记忆库、治理、设置页面应偏工具型：

- 高信息密度
- 清晰表格/列表
- 明确状态
- 操作稳定
- 右侧详情面板

### 图谱页面

图谱页可以更有情绪和传播感：

- 更沉浸
- 更动态
- 更像“我的记忆宇宙”
- 但不能变成不可操作的炫技画面

## 8. 组件方向

V1 推荐组件风格：

- 左侧导航：窄栏，图标 + 中文标签。
- 顶部搜索：全局命令栏风格。
- 快捷导入：按钮组或命令菜单。
- 状态标签：短文字 + 小色块。
- 表格：紧凑但不拥挤。
- 详情面板：右侧固定，可折叠。
- 弹窗：只用于确认、删除、授权、模型配置。
- 图谱节点：按类型使用不同形状、颜色或大小。

## 9. 不要做

- 不要首页先放传统仪表盘，让图谱变成二级页面。
- 不要把 `chunk`、`embedding` 直接暴露给普通用户。
- 不要让新用户第一步就必须配置大模型。
- 不要在无数据时伪造图谱。
- 不要把状态藏到详情页。
- 不要让删除操作只有一个“删除”按钮。
- 不要把图谱做成只能看不能搜的装饰。
