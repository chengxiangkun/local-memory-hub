# 信息架构：Local Memory Hub V1

## 1. 目标

本文件定义 Local Memory Hub V1 的页面结构、导航、页面职责、核心状态和关键用户路径。

设计目标：

- 用户能低成本把资料导入系统。
- 用户能看懂资料是否已保存、解析、入记忆。
- 用户能搜索、问答、追溯来源。
- 用户能治理污染和删除派生产物。
- 用户能通过图谱探索关系。
- 用户能配置模型和外部 AI 接入。

## 2. 设计原则

### 2.1 图谱优先，不做营销页

打开应用后默认进入图谱首页，不做欢迎大屏、品牌故事或营销式 hero。图谱首页要像 Obsidian 一样，第一眼就让用户看到“我的资料正在形成网络”。

### 2.2 状态必须可见

这些状态必须在关键页面直接可见：

- 导入状态
- 解析状态
- 入记忆状态
- 追溯状态
- 污染状态
- 外部调用状态

状态不应该藏在详情页最深处。用户必须一眼知道资料现在能不能被 AI 使用。

### 2.3 源资料和记忆分离

页面上明确区分：

- 源资料：用户保存的原始文件、链接、截图、外部文档。
- 记忆：AI 可以检索和使用的文本片段、摘要、向量索引、图谱节点。

这两个视角不要混成一个列表。

### 2.4 默认简单，高级能力折叠

用户常用操作要直接显示：

- 导入
- 搜索
- 问答
- 查看状态
- 打开源文件
- 标记污染

高级操作可放在更多菜单或详情页：

- 删除向量
- 删除图谱关系
- 重建向量索引
- 重建图谱
- 查看解析日志
- 查看模型调用日志

### 2.5 图谱要有传播感，但不能牺牲可用性

图谱页可以更炫酷，但必须保留：

- 搜索
- 节点定位
- 节点详情
- 关系原因
- 点回源文件
- 污染过滤

## 3. 顶层导航

建议采用左侧主导航 + 顶部全局搜索。

```text
图谱首页
工作台
Inbox
源资料
记忆库
搜索/问答
治理
外部接入
设置
```

### 3.1 导航说明

| 页面 | 目的 | 是否 V1 必须 |
| --- | --- | --- |
| 图谱首页 | 默认首页，以 Obsidian 风格图谱展示记忆网络、搜索和快速导入 | 是 |
| 工作台 | 展示全局状态、任务概览和治理提醒 | 是 |
| Inbox | 管理新进入但未完全处理的资料 | 是 |
| 源资料 | 管理原始文件和外部来源 | 是 |
| 记忆库 | 查看 AI 可用记忆和派生产物 | 是 |
| 搜索/问答 | 搜索资料、无模型兜底、有模型问答 | 是 |
| 治理 | 查看污染、隔离、删除和恢复 | 是 |
| 外部接入 | MCP/API/Codex 接入 | 是 |
| 设置 | 数据目录、模型、连接器、隐私 | 是 |

## 4. 全局布局

### 4.1 桌面端布局

```text
┌──────────────────────────────────────────────────────────┐
│ 顶部栏：全局搜索 / 快捷导入 / 模型状态 / 任务状态          │
├──────────────┬─────────────────────────────┬─────────────┤
│ 左侧导航      │ 主内容区                     │ 右侧详情面板 │
│              │                             │ 可折叠       │
└──────────────┴─────────────────────────────┴─────────────┘
```

### 4.2 右侧详情面板

右侧详情面板用于显示当前选中对象：

- 源资料详情
- 记忆详情
- 图谱节点详情
- 解析任务详情
- 污染原因
- 外部调用日志

这样可以减少页面跳转，让工作流更像桌面工具。

### 4.3 移动或窄屏

V1 主目标是桌面端。窄屏时：

- 左侧导航折叠成图标或抽屉。
- 右侧详情面板变成全屏弹层。
- 表格降级为列表卡片。

## 5. 页面设计

## 5.0 首次启动引导

### 页面目标

安装后引导用户完成最小可用配置，并让用户立刻看到第一条资料如何进入系统。

### 引导步骤

```text
步骤 1：选择数据目录
步骤 2：配置大模型，可跳过
步骤 3：随便导入一段文本
步骤 4：系统保存、解析并生成第一条记忆
步骤 5：进入图谱首页，看到第一个节点
```

### 步骤 2：配置大模型

该步骤引导用户配置主流模型或自定义接口：

- OpenAI
- Anthropic
- Gemini
- DeepSeek
- 通义千问
- 豆包
- 智谱
- Kimi
- Ollama / LM Studio
- 自定义 OpenAI-Compatible Endpoint

必须允许跳过。跳过后提示：

```text
你可以先不配置大模型。搜索和本地资料管理仍然可用，之后可在设置中补充。
```

### 步骤 3：导入一段文本

引导页提供简单文本框：

```text
随便粘贴一段你最近看到的内容，系统会把它保存为第一条源资料。
```

操作：

- 粘贴文本
- 使用示例文本
- 跳过

导入后系统展示：

- 已保存
- 解析成功
- 已入记忆
- 已生成图谱节点

引导完成后进入图谱首页，并默认聚焦刚导入的第一条资料。

## 5.1 图谱首页

### 页面目标

作为应用默认首页，以 Obsidian 风格图谱展示用户的资料和记忆关系，同时提供搜索、快捷导入和节点追溯。

### 核心模块

- 图谱画布
- 图谱搜索
- 快捷导入
- 节点详情面板
- 模型状态
- 任务状态
- 污染节点开关

### 默认展示

- 最近导入的源资料节点
- 最近生成的主题节点
- 最近问答引用过的记忆节点
- 高关联度的想法/项目节点
- 当前污染节点默认隐藏

### 空状态

如果用户还没有任何资料：

```text
你的记忆图谱还没有开始生长。

[导入一段文本] [上传文件] [粘贴链接] [配置模型]
```

空状态仍然保留图谱画布的视觉氛围，但不要假装已有数据。

## 5.2 工作台

### 页面目标

让用户看到系统当前状态、解析任务、最近资料、治理提醒和外部调用状态。

### 核心模块

- 正在处理任务
- 最近源资料
- 最近记忆召回
- 污染提醒
- 图谱亮点
- MCP 状态

### 主要操作

- 查看处理任务
- 查看失败项
- 查看污染提醒
- 进入搜索/问答
- 打开图谱

### 空状态

首次启动时显示：

- 选择数据目录
- 导入第一个文件
- 配置模型，允许跳过
- 启用 MCP，允许稍后

不要用大段说明文字。用 3-4 个清晰操作入口即可。

## 5.2 Inbox

### 页面目标

承接所有新导入资料，显示处理进度和需要用户处理的事项。

### 列表字段

- 来源平台
- 导入方式
- 导入时间
- 解析状态
- 入记忆状态
- 污染状态
- 需要操作

### 筛选

- 全部
- 等待解析
- 解析中
- 解析失败
- 等待入记忆
- 需要授权
- 需要上传兜底
- 已污染

### 主要操作

- 查看详情
- 重新解析
- 授权外部平台
- 上传兜底文件
- 加入记忆
- 忽略
- 标记污染

作为本地源文件和外部来源的可信管理器。

### 视图

- 文件夹视图
- 日期视图
- 类型视图

- 状态视图

### 列表字段

- 文件名/标题
- 文件夹
- 日期
- 类型
- 来源
- 源路径/源链接
- 解析状态
- 是否已入记忆
- 追溯状态

### 主要操作

- 打开源文件
- 预览
- 重新解析
- 加入记忆
- 从记忆中移除
- 标记污染
- 删除
- 高级删除

### 详情面板

显示：

- 基础信息

- 原始路径或 URL

### 视图

- 按最近使用
- 按状态
- 按污染/隔离

### 列表字段

- 记忆标题
- 来源资料
- 文本片段数量

- 图谱状态
- 摘要状态
- 最近被使用
- 污染状态

### 主要操作

- 查看文本片段
- 打开源资料
- 从向量中移除
- 从图谱中移除
- 重建向量索引
- 重建图谱关系
- 标记污染

## 5.5 搜索/问答

### 页面目标

统一承接搜索和问答。

### 模式

- 搜索模式
- 问答模式
- 无模型兜底模式

### 搜索输入

输入框支持：

- 关键词搜索
- 自然语言问题
- 来源过滤
- 日期过滤
- 类型过滤
- 是否包含污染内容

### 无模型状态

如果没有可用模型：

- 显示“当前未启用大模型，已为你展示相关资料”。
- 返回源资料、命中片段、相关记忆、相关图谱节点。
- 提供“配置模型后生成答案”入口。

### 有模型状态

回答区显示：

- 答案
- 引用来源
- 使用到的文本片段
- 图谱扩展节点
- 模型信息
- 标记错误引用

## 5.6 图谱

### 页面目标

让用户探索资料、主题、想法、项目、人物之间的关系，并形成视觉传播点。

### 布局

```text
┌──────────────────────────────────────────────┐
│ 图谱工具栏：搜索 / 筛选 / 模式 / 导出         │
├───────────────────────────────┬──────────────┤
│ 图谱画布                       │ 节点详情      │
│                               │ 关系解释      │
└───────────────────────────────┴──────────────┘
```

### 工具栏

- 搜索节点
- 来源筛选
- 节点类型筛选
- 关系类型筛选
- 污染节点开关
- 时间线模式
- 聚类模式
- 导出快照

### 节点详情

显示：

- 节点名称
- 节点类型
- 来源资料
- 关联记忆
- 相邻节点
- 关系原因
- 污染状态
- 打开源资料
- 从图谱移除

### 视觉方向

图谱页可以比其他页面更有视觉张力：

- 深色画布可选。
- 节点有层级和类型差异。
- 选中节点产生聚焦光晕或路径高亮。
- 聚类之间有明确空间关系。

但不要牺牲可读性。文字、状态和操作必须清楚。

## 6. 核心用户路径

## 6.1 导入到搜索

```text
工作台
  ↓
上传文件 / 粘贴链接
  ↓
Inbox
  ↓
源资料库查看状态
  ↓
解析成功
  ↓
已入记忆
  ↓
搜索/问答
  ↓
打开源文件
```

## 6.2 解析失败到大模型兜底

```text
源资料导入
  ↓
本地解析失败
  ↓
状态显示失败原因
  ↓
根据策略调用大模型
  ↓
兜底成功
  ↓
保存解析改进产物
  ↓
入记忆
```

## 6.3 问答到污染治理

```text
搜索/问答
  ↓
生成带引用答案
  ↓
用户发现错误引用
  ↓
标记污染
  ↓
进入治理页面
  ↓
污染内容从搜索/问答/图谱中排除
```

## 6.4 图谱到源资料

```text
图谱
  ↓
搜索节点
  ↓
聚焦节点
  ↓
查看关系原因
  ↓
打开源资料
```

## 6.5 外部 AI 调用

```text
外部接入
  ↓
启用 MCP
  ↓
复制 Codex 配置
  ↓
外部工具调用 memory.search
  ↓
返回结构化上下文
  ↓
应用内查看调用日志
```

## 7. 状态表达

### 7.1 状态分组

| 状态组 | 示例 |
| --- | --- |
| 导入 | 已保存、导入失败、需要授权、需要上传兜底 |
| 解析 | 等待解析、解析中、解析成功、解析失败、兜底中 |
| 记忆 | 未入记忆、入记忆中、已入记忆、部分入记忆、入记忆失败 |
| 治理 | 干净、疑似污染、已污染、已隔离、已删除 |
| 追溯 | 可追溯、源文件缺失、外部链接失效、派生产物残留 |

### 7.2 UI 表达建议

- 状态使用短标签，不用长句。
- 失败状态必须可点击查看原因。
- 污染和隔离状态要有明显但不过度刺眼的标识。
- “已入记忆”不要和“解析成功”混用。
