# Local Memory Hub 架构说明

## 总体结构

Local Memory Hub 是本地优先的个人记忆系统：

- Web UI：图谱、导入、源资料、问答、治理、设置。
- Local API：导入、解析、向量、图谱、模型 Provider、连接器、治理。
- MCP-like Server：供 Codex 等外部 AI 工具读取本地记忆。
- 本地数据目录：保存源资料、解析结果、配置、索引、日志和 trash。

## 数据目录

核心目录：

- `raw/`：源资料快照。
- `extracted/`：解析后的文本和音频。
- `database/`：SQLite 与本地数据文件。
- `config/`：本地模型、连接器、文件夹配置。
- `logs/`：模型调用和外部调用日志。
- `trash/`：软删除源文件。

## 导入系统

所有入口统一为 `ImportRequest`：

- text
- file
- url
- external connector

导入器通过 `ImporterRegistry` 选择具体实现，避免把飞书、腾讯文档、视频平台等逻辑写进同一个函数。

## 解析管线

解析顺序：

1. 本地解析优先。
2. 本地失败时可调用模型兜底。
3. 模型兜底结果保存为改进样例。
4. 生成文本片段。
5. 写入向量索引和图谱节点。

## 搜索和问答

问答先检索本地资料，再把上下文交给所选模型。没有模型时返回本地兜底结果。问答对话可以沉淀为新的记忆，并由记忆整理 Agent 维护用户习惯画像。

## 污染治理

污染治理按源资料级别先实现：

- 隔离源资料。
- 同步隔离文本片段、向量和图谱节点。
- 支持恢复。
- 删除源文件时可选择是否删除派生数据。

后续可扩展到片段级治理。

## 外部连接器

飞书和腾讯文档先通过连接器状态层接入：

- 保存平台、账号、同步模式和删除策略。
- 支持同步任务登记。
- 真实 OAuth/API Adapter 后续接入同一接口。

## 外部 AI 接入

MCP-like 服务暴露本地记忆搜索和上下文获取能力。默认只监听本地地址，所有外部调用写入日志，且默认不返回隔离内容。
